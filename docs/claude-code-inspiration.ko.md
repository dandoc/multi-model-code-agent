# Claude Code 영감 문서

## 이 문서의 목적

이 프로젝트는 갑자기 생긴 것이 아닙니다. Claude Code의 워크플로우와 외부에서 관찰 가능한 제품 동작에서 드러나는 방법론을 연구하고, 그 아이디어를 여러 LLM에 대응할 수 있는 별도 코드베이스에서 검증해보려는 더 큰 프로젝트의 일부입니다.

장기 목표는 이 CLI 하나에만 있지 않습니다.

- Claude Code의 어떤 아이디어가 실제로 강한지 이해하기
- 그 아이디어를 직접 구현하면서 검증하기
- 로컬 모델과 원격 모델이 섞인 환경에 맞게 적응시키기
- 특정 hosted 모델 하나가 아니라 여러 LLM에 잘 맞는 vibe-coding tool을 만들기

`multi-model-code-agent`는 그 더 큰 목표 안에서 실전 테스트베드 역할을 하는 CLI입니다.

## 이 프로젝트가 차용하는 것

이 프로젝트는 Claude Code를 `방법론` 차원에서 강하게 참고합니다.

핵심은 다음과 같습니다.

- REPL 중심 코딩 흐름
  - 한 루프 안에서 질문, 파일 탐색, 수정, 실행, 후속 질문을 이어가는 방식
- 툴 기반 에이전트 루프
  - 모델이 직접 파일을 고쳤다고 주장하는 대신, 명시적인 도구를 요청하고 결과를 다시 받는 구조
- 위험 작업에 대한 사람 승인
  - 셸 실행과 파일 수정은 기본적으로 보이고, 확인받고, 되돌릴 수 있어야 한다는 관점
- 근거 기반 저장소 분석
  - 모델 추측에만 기대지 않고 실제 파일과 결정론적 helper를 함께 쓰는 방식
- 세션 저장과 런타임 사용성
  - 코딩 작업을 일회성 프롬프트가 아니라 이어지는 세션으로 보는 관점

이런 부분들은 충분히 연구하고 재사용할 가치가 있다고 판단했습니다.

## 왜 그대로 복제하지 않는가

이 저장소의 목표는 Claude Code의 내부 구현을 그대로 재현하는 것이 아닙니다.

그 이유는 분명합니다.

- 목표 환경이 다릅니다
  - 이 프로젝트는 `Ollama`, `OpenAI-compatible` 백엔드, `Codex CLI`를 함께 다뤄야 합니다
- 로컬 모델은 hosted 대형 모델과 다르게 동작합니다
  - `Qwen`, `Gemma` 같은 경로는 grounding, 응답 정규화, 실패 복구가 더 중요합니다
- 설계 목표가 portability입니다
  - 모델과 공급자가 바뀌어도 구조가 버텨야 합니다
- 구현을 직접 통제하고 싶었습니다
  - smoke, path safety, session/profile, retry, preflight 같은 층을 프로젝트 요구에 맞게 조절해야 했습니다

그래서 이 프로젝트는 Claude Code의 사용 방식과 안전장치 철학은 받아들이되, 구현은 별도로 다시 만들고 있습니다.

## 이 저장소에서 의도적으로 다른 부분

이 CLI는 backing model이 고정되지 않았을 때 더 중요해지는 문제를 중심에 둡니다.

예를 들면:

- 교체 가능한 provider adapter
  - `Ollama`, `OpenAI-compatible`, `Codex CLI`
- provider별 failure diagnosis와 retry 규칙
- provider/model/profile 전환 전 preflight
- 모델마다 다른 응답 형식에 대한 normalization
- live smoke를 통한 실공급자 검증
- session/profile/runtime 상태를 명시적으로 다루는 UX

이건 부수적인 디테일이 아니라, “어떤 LLM을 붙여도 잘 작동하는 도구”를 만들기 위한 핵심입니다.

## 더 큰 프로젝트의 목표

이 더 큰 프로젝트는 다음처럼 요약할 수 있습니다.

1. Claude Code의 방법론을 진지하게 연구한다.
2. 쓸 만한 부분을 별도 코드베이스에 구현한다.
3. 실제 개발 작업에 계속 적용해본다.
4. 모델이 바뀌어도 남는 원칙만 추려낸다.
5. 그 결과를 바탕으로 더 강한 멀티 LLM vibe-coding tool을 만든다.

이 관점에서 보면 이 CLI는 최종 목적지 전체가 아니라, 그 과정 안에 있는 실제 산출물입니다.

## 현재 가장 정확한 해석

이 저장소를 가장 짧고 정확하게 설명하면 다음과 같습니다.

> Claude Code에서 영감을 받은 멀티 모델 코딩 에이전트 CLI이자, 여러 LLM에 잘 맞는 vibe-coding tool을 만들기 위한 연구·검증용 실전 테스트베드

즉 이 프로젝트는:

- Claude Code의 방법론에서 영감을 받았고
- 구현은 독립적으로 만들었으며
- 멀티 모델 portability를 중요하게 보고
- 이론이 아니라 실제 개발 작업으로 계속 검증하는 도구입니다
